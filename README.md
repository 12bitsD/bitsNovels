# bitsNovels

> **AI 辅助长篇小说创作平台**

------

## 这是什么？

bitsNovels 是一个专为长篇小说作者打造的**智能写作工作室**。

每天写作 8 小时以上的职业作者、需要管理 50+ 角色和跨卷伏笔的世界观型作者——你们面对的不仅是"写不出来"，还有**设定前后矛盾、人物关系混乱、知识库无法追踪**这些隐藏的敌人。

bitsNovels 的核心差异：**AI 自动识别正文中的人物、地点、道具、势力、伏笔**，构建可追踪、可视化的知识库。写作时知识库在旁边，设定有冲突？AI 提醒你。伏笔埋了没回收？AI 追踪你。

作者始终掌控写作的主导权，AI 是助手，不是替代者。

------

## 核心功能

bitsNovels 按功能分为 6 个模块（Epic）：

| Epic | 功能 | 一句话说明 |
|------|------|-----------|
| EPIC1 · 项目管理 | 登录注册、项目列表、新建项目、卷章目录、写作目标、回收站、归档、导入作品 | 所有创作都在项目里 |
| EPIC2 · 知识库 | AI 自动识别角色/地点/道具/势力/伏笔，关系图谱，一致性检查，全局搜索 | 让 AI 帮你管理世界观 |
| EPIC3 · 写作工作台 | 专业富文本编辑器，专注模式，版本快照，内容批注，查找替换，写作统计 | 流畅、沉浸、不丢稿 |
| EPIC4 · AI 辅助写作 | AI 续写、润色、扩写缩写、对话生成、大纲建议、名字生成器、写作建议 | 卡文时给你方向，不给你答案 |
| EPIC5 · 导出备份 | DOCX/TXT/PDF/EPUB 导出，自动备份，项目恢复 | 写完的作品随时拿走 |
| EPIC6 · 系统设置 | 用户资料、界面偏好、快捷键、通知管理、存储管理 | 让工具适配你，不是你适应工具 |

------

## 设计主题

**复古科学 × 数字羊皮纸**

bitsNovels 的界面像一本被数字化的老科学家野外笔记——温暖的琥珀色调、精确的线条装饰、羊皮纸质感背景，加上现代极简的功能界面。

为什么选这个风格？因为你每天要看它 8 小时。它必须**耐看、不疲劳、有质感**。

主题支持明暗两套，自动跟随系统。

## 目录结构

```
bitsNovels/
├── INDEX.md              ← 你在这里，路由总表
├── README.md             ← 你现在看的，项目介绍
├── USER_STORIES.md       ← 完整需求文档，单一真相源
│
├── docs/                 ← 项目管理文档
│   ├── CONTEXT.md        ← 项目上下文、Epic 顺序
│   ├── CHANGELOG.md      ← 变更记录
│   └── SPRINT_LOG.md     ← Sprint 跟踪
│
├── specs/                ← 需求规格
│   └── EPIC/             ← 按 Epic 拆分
│       ├── INDEX.md      ← Epic 路由表
│       ├── EPIC1/        ← 项目管理（FE/BE/context）
│       ├── EPIC2/        ← 知识库（FE/BE/context）
│       ├── EPIC3/        ← 写作工作台（FE/BE/context）
│       ├── EPIC4/        ← AI 写作（FE/BE/context）
│       ├── EPIC5/        ← 导出备份（FE/BE/context）
│       └── EPIC6/        ← 系统设置（FE/BE/context）
│
├── design/               ← 设计规格
│   ├── FRONTEND.md       ← 前端视觉规范
│   └── BACKEND.md        ← 后端架构规范
│
└── process/              ← 流程规格
    ├── TDD.md            ← TDD 工作流
    └── CONSTRAINTS.md    ← NFR 约束表
```

**核心设计原则**：每个 Epic 的前端（FE）和后端（BE）拆分文件，共同引用 `context.md` 作为数据约定。Agent 每次任务最多读 4 个文件。

## 如何参与？

如果你想帮忙开发或提建议：

1. 先读 `INDEX.md` 了解整体架构
2. 再读 `docs/CONTEXT.md` 了解项目阶段和 Epic 顺序
3. 根据你想参与的方向，读对应的 Epic 文件（`specs/EPIC/N/FE.md` 或 `BE.md`）

技术栈决策完成后，我们会在 `docs/SPRINT_LOG.md` 记录，并开放第一批开发任务。
