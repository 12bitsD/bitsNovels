# bitsNovels · 非功能需求约束

> 本文档从各 Epic spec 文件中提取所有硬性数值约束，按类别组织为 NFR 表。
> V1 版本 | 2026-03-20

------

## 核心约束速查

bitsNovels V1 的硬性约束按**性能 / 可靠性 / 兼容性 / 存储 / AI** 五类组织。所有开发实现不得突破这些上限。

------

## 1. 性能约束

| 约束项 | 具体值 | 来源 US | 备注 |
|--------|--------|---------|------|
| 自动保存防抖 | 3 秒 | US-3.1 AC#3 | 用户停止输入后 3 秒触发保存 |
| Parser 防抖（同一章节） | 60 秒 | US-2.1 AC#1 | 60 秒内不重复加入解析队列 |
| Parser 单章超时 | 60 秒 | US-2.1 AC#5 | 超时后自动重试一次，再失败标记失败 |
| 撤销栈深度 | ≥ 200 步 | US-3.1 AC#2 | 涵盖文本操作、格式变更、段落操作 |
| 撤销栈合并 | 500ms 停顿或换行 | US-3.1 AC#2 | 连续输入以停顿分割为一个撤销步骤 |
| 搜索防抖 | 300ms | US-2.9 AC#1 | 搜索框输入 ≥ 2 字后触发 |
| 自动保存重试 | 3 次 | US-3.1 AC#3 | 连续失败后弹出手动保存提示 |
| 编辑器状态更新 | 实时 | US-3.1 AC#5 | 状态栏实时显示字数 |
