# bitsNovels · 前端视觉设计规范

> "复古科学 × 数字羊皮纸" — 融合复古实验室美学与现代写作体验。

------

## 1. 设计主题

bitsNovels 的视觉语言是**复古科学 × 数字羊皮纸**。

核心理念：像一位 19 世纪科学家的皮革野外笔记簿被数字化——厚重的羊皮纸质感、精确的线条装饰、温暖的琥珀色调，搭配现代写作界面的极简功能感。

目标用户每天写作 8 小时以上，界面需要：**耐看**（不疲劳）、**沉浸**（不干扰）、**专业**（有质感）。

## 3. 字体系统

### 3.1 编辑器字体（正文写作）

| 字体选项 | 使用场景 | 推荐理由 |
|----------|----------|----------|
| 系统默认 | 通用 | 各平台最优渲染 |
| 宋体（SimSun） | 中文长篇阅读 | 传统阅读体验 |
| 黑体（SimHei） | 章节标题 | 清晰有力 |
| 楷体（KaiTi） | 引用/特殊段落 | 典雅 |
| 等宽字体（Courier/New York） | 字数统计区 | 数据精确感 |

**默认配置**：正文宋体 / 标题黑体 / 统计等宽

### 3.2 界面字体

| 用途 | 推荐字体 |
|------|----------|
| 界面 UI | system-ui, -apple-system, "PingFang SC", "Microsoft YaHei" |
| 代码/技术文本 | "SF Mono", "Cascadia Code", monospace |

### 3.3 字号与行距

| 编辑器正文 | 16px（可调 14~24px） | 1.75 倍（可调 1.5/1.75/2.0） |

| 章节标题 H2 | 20px | - |

## 5. 组件规范

### 5.1 按钮

| 类型 | 样式 | 使用场景 |
|------|------|----------|
| 主按钮 | 琥珀金底色 + 深棕文字 | 主要操作（如"创建项目"） |
| 次按钮 | 透明底 + 琥珀边框 | 次要操作 |
| 危险按钮 | 深红底色 | 删除、永久移除 |
| 文字按钮 | 无底色，琥珀色文字 | 辅助操作（如"取消"） |
| 禁用态 | 50% 透明度 | 不可点击 |

### 5.2 卡片（项目卡片、知识库卡片）

- 羊皮纸质感背景
- 左侧 4px 琥珀色边线
- 悬停：微微上浮 + 阴影加深
- 卡片内文字省略规则：标题单行截断，描述两行截断

### 5.3 模态对话框

- 居中弹出
- 遮罩层：`rgba(44,36,22,0.5）`
- 右上角关闭按钮（×）
- 标题区 + 内容区 + 操作按钮区，三段式
- 危险操作确认需二次输入

### 5.4 表单

- 标签在输入框上方
- 输入框：琥珀色聚焦边框（2px）
- 错误态：红色边框 + 错误提示文字在下方
- 必填标记：琥珀色圆点

### 5.5 通知 Toast

- 出现在右上角
- 自动消失时间：3 秒（可配置）
- 类型色彩条（左侧 4px）：成功绿/警告黄/错误红/信息蓝
- 可手动关闭

### 5.6 状态栏（编辑器底部）

- 高度 32px
- 背景色略深于编辑器
- 低对比度文字（次级文字色）
- 显示内容：章节字数、保存状态、今日进度（若设置了目标）

### 5.7 批注高亮

| 状态 | 背景色 |
|------|--------|
| 活跃批注 | `rgba(200,160,60,0.25)` 淡琥珀 |
| 已解决批注 | `rgba(150,140,120,0.15)` 浅灰 |
| AI 生成内容 | 虚线下划线 |

## 6. 专注模式

- 全屏沉浸，无任何 UI 干扰
- 仅保留编辑区（居中，宽度不变）+ 底部状态栏（低对比度）
- 背景色跟随主题：浅色为 `#F5F0E8`，深色为 `#1A1714`
- 进入方式：`F11` 或工具栏按钮
- 退出：`Esc` 或鼠标移到顶部触发退出按钮

## 7. 动画规范

### 7.1 微交互

| 场景 | 动画 |
|------|------|
| 保存状态变化 | 文字淡入淡出切换，无弹跳 |
| 进度条达成 | 绿色闪烁一次 + 对勾图标弹出，300ms |
| 目标达成 | 进度条变绿 + 轻微放大，500ms |
| Toast 出现 | 从右侧滑入，200ms ease-out |
| Toast 消失 | 淡出，150ms |
| 面板展开/折叠 | 宽度过渡，200ms ease-in-out |
| 解析状态图标 | 旋转动画（解析中），无限循环 |

### 7.2 不使用动画的场景

- 编辑器输入时（零干扰）
- 自动保存状态（无视觉打断）

## 8. 无障碍规范

| 要求 | 标准 |
|------|------|
| 最小对比度 | 正文文字 ≥ 4.5:1（WCAG AA） |
| 大文字对比度 | 标题 ≥ 3:1 |
| 键盘导航 | 所有交互元素可 Tab 聚焦 |
| 聚焦指示器 | 琥珀色 2px 边框 |
| 屏幕阅读器 | ARIA 标签覆盖所有图标按钮 |
| 减少动画 | 尊重 `prefers-reduced-motion` |

## 9. 技术栈（已确认）

| 决策项 | 已确认方案 | 备注 |
|--------|-----------|------|
| 前端框架 | **React + TypeScript** | AI 生成质量最稳，npm 生态最大 |
| 编辑器引擎 | **TipTap 核心**（免费开源） | Diff/批注自研（`diff-match-patch` + 自定义 Mark） |
| CSS 方案 | **Tailwind CSS** | AI 写起来最快，与 React 集成最佳 |
| 前后端类型同步 | **openapi-typescript** | FastAPI 自动生成 openapi.json，前端构建时生成 TS 类型 |

### TipTap 选型依据

详见 `docs/decisions/tech-stack.md`。

**选 TipTap 不选 Slate 的理由**：
- ProseMirror 底层更成熟，生产案例更多（Notion/Excalidraw 等）
- `insertContentAt` 命令式 API 对 AI 流式插入更友好
- `UndoRedo` 可配置 depth + newGroupDelay（500ms 合并），满足 US-3.1
- `CharacterCount` 原生扩展免费
- 官方仍标 beta，API 变化风险高于 TipTap

**Markdown 实时渲染（类 Typora）**：
- 集成 `@tiptap/extension-markdown`（免费开源）
- 输入 Markdown 语法自动渲染为富文本：`**文字**` → 加粗，`## ` → H2，`---` → 分隔线
- 编辑器内部存储富文本 JSON，Markdown 是输入快捷方式而非存储格式
- 导出 Markdown 时将富文本 JSON 反向转换为 Markdown 字符串

**付费扩展不采用的理由**：
- Comments（批注）和 Snapshot Compare（Diff 对比）均为付费私有 Extensions
- 两者均可通过开源方案替代：Diff → `diff-match-patch`；批注 → 自定义 Mark + React 侧边栏
- 额外研发成本约 3-5 个工作日，属于可接受范围

**AI 流式输出实现方案**（免费）：
- `editor.commands.insertContentAt()` 按位置插入 token
- 自定义 Mark 标记 AI 生成内容（浅灰底 + 虚线下划线）
- "采纳"→ 移除 Mark 合并正文；"放弃"→ 删除该 Mark 范围内容

视觉设计规范本身**不依赖具体框架**，上述颜色、字体、间距系统在任何框架中均可复现。
