# Epic 1 · 项目管理 · 前端

> 标记说明：`[T]` = 前端独立职责，`[M]` = 需要 FE/BE 联调的交互。

## US-1.1 用户注册与登录

**依赖**：无
**前置 Epic**：无

### AC#1 注册方式

[T] 1. 注册页提供邮箱、密码、确认密码输入框与提交按钮。
[T] 2. 密码输入实时校验：长度至少 8 位，且同时包含大写字母、小写字母、数字；不满足时显示对应错误提示。
[M] 3. 表单校验通过后调用 `POST /api/auth/register`，成功后显示“注册成功，请查收验证邮件”。
[M] 4. 验证邮件回跳前端验证结果页时，展示“验证成功 / 链接已过期”的状态反馈，并提供重新发送入口。
[M] 5. 未验证邮箱的用户登录成功后，页面顶部显示“请验证您的邮箱以确保账号安全”提示条，并支持点击“重新发送”。

### AC#2 第三方登录

[T] 1. 登录页展示 Google、GitHub OAuth 按钮，包含 loading 与失败重试状态。
[M] 2. 点击第三方登录后跳转授权页，回跳后处理成功、取消授权、授权失败三种结果。
[M] 3. 首次第三方登录成功后，前端展示第三方昵称作为默认昵称，并允许用户稍后在个人资料页修改。
[M] 4. 同邮箱账号被自动绑定时，前端直接进入已绑定账号，不要求用户额外确认。

### AC#3 登录与会话

[T] 1. 登录表单提供邮箱、密码、记住我、提交按钮、错误提示与 loading 状态。
[M] 2. 提交后调用 `POST /api/auth/login`，请求体包含 `rememberMe`。
[M] 3. 登录成功后写入会话信息并跳转到项目列表页。
[M] 4. 会话过期、被重置密码挤下线或手动登出后，前端统一清理本地登录态并跳回登录页。

### AC#4 找回密码

[T] 1. 登录页提供“忘记密码”入口，跳转到邮箱填写页。
[M] 2. 提交邮箱后调用 `POST /api/auth/forgot-password`，统一显示“如邮箱已注册，将收到重置邮件”。
[M] 3. 重置链接页面提供新密码、确认密码、强度校验与提交状态。
[M] 4. 调用 `POST /api/auth/reset-password` 成功后，显示“密码已重置，请重新登录”，并要求用户重新建立会话。

## US-1.2 项目列表与仪表盘

**依赖**：US-1.1
**前置 Epic**：Epic 1

### AC#1 项目列表

[M] 1. 登录后默认调用 `GET /api/projects`，按最后编辑时间倒序渲染项目列表。
[T] 2. 项目卡片 / 列表项展示项目名称、封面色块、类型标签、总字数、章节数、最后编辑时间。
[T] 3. 支持卡片视图与列表视图切换，并保留当前用户的视图偏好。

### AC#2 排序与筛选

[T] 1. 提供排序下拉：最后编辑时间、创建时间、项目名称（A-Z）、总字数。
[T] 2. 提供筛选项：类型、状态，以及项目名称模糊搜索框。
[M] 3. 排序、筛选、搜索变化后触发重新请求或本地重排，并保持当前筛选状态可恢复。
[T] 4. 无匹配数据时展示“未找到匹配项目”结果态，而不是空白页面。

### AC#3 快捷操作

[T] 1. 每个项目卡片右上角提供更多菜单（打开、项目设置、归档、删除）。
[M] 2. 点击项目卡片时进入写作工作台，并根据后端返回的 `lastEditedChapterId` 定位到上次编辑章节。
[T] 3. 页面顶部提供醒目的“+ 新建项目”按钮，点击进入 US-1.3 创建流程。

### AC#4 空状态

[T] 1. 新注册用户无项目时，展示居中空状态页，包含标题、3 条 Feature 卡片、主 CTA。
[T] 2. “新建项目”CTA 直接跳转到创建项目流程。

## US-1.3 创建新项目

**依赖**：US-5.4（知识库模板导出，仅在模板导入场景需要）
**前置 Epic**：Epic 5（可选联动）

### AC#1 创建流程（分步表单）

[T] 1. 创建流程采用三步向导，显示步骤条、返回、下一步、创建、跳过操作。
[T] 2. Step 1 提供项目名称、类型、题材标签、简介表单，并显示必填、字数上限、标签上限提示。
[T] 3. Step 2 提供“空白项目 / 从模板开始”二选一，并在模板模式下展示结构模板卡片。
[M] 4. Step 3 支持上传知识库 JSON 模板文件，校验文件大小，展示条目总数和按类型分布摘要。
[T] 5. Step 3 支持全选 / 按类型勾选导入条目，并允许点击“跳过”。
[M] 6. 导入过程中显示进度条，完成后 toast 提示“已导入 N 条知识库条目”。
[M] 7. 点击“创建”后调用项目创建接口，成功后跳转到项目仪表盘或写作工作台。

### AC#2 校验

[T] 1. 项目名称输入时先做 `trim`，为空时阻止进入下一步或提交。
[T] 2. 前端校验项目名称长度不超过 50、题材标签最多 5 个、简介不超过 500 字。
[T] 3. 知识库模板导入文件仅接受 JSON，大小不超过 20MB。
[M] 4. 后端返回同一用户下项目重名冲突时，在名称字段下显示“已有同名项目，请修改名称”，并保留当前表单数据。

## US-1.4 项目设置

**依赖**：US-1.3
**前置 Epic**：Epic 1

### AC#1 设置入口

[T] 1. 写作工作台顶部菜单栏提供“项目设置”齿轮入口。
[T] 2. 项目列表卡片更多菜单中提供“项目设置”入口。
[T] 3. 设置页面以 Tab 组织：基本信息、写作目标、AI 配置、备份与恢复。
[M] 4. 进入设置页时拉取项目详情与只读统计数据。

### AC#2 基本信息 Tab

[T] 1. 基本信息 Tab 复用 US-1.3 的字段与校验规则。
[T] 2. 页面底部展示只读信息：创建时间、总字数、总章节数、知识库条目数。
[M] 3. 点击“保存”后调用更新接口，成功后 toast“项目信息已更新”，失败时保留当前编辑内容。

### AC#3 危险操作区

[T] 1. 页面底部危险区域展示“归档项目”“删除项目”操作，并与常规表单视觉区分。
[T] 2. 删除项目时弹出二次确认对话框，要求输入项目名称，完全匹配前删除按钮保持禁用。
[M] 3. 删除成功后跳回项目列表，并清理该项目相关的本地缓存、最近访问记录和当前路由状态。

## US-1.5 卷章目录管理

**依赖**：US-1.3
**前置 Epic**：Epic 1

### AC#1 卷管理

[T] 1. 大纲视图以树形结构展示卷与章节，支持展开 / 折叠。
[T] 2. 提供“+ 新建卷”入口，弹窗或行内输入都需要校验卷名必填且不超过 30 字。
[T] 3. 支持双击卷名或右键菜单重命名，并提供卷简介编辑表单。
[M] 4. 删除卷时，如卷内存在章节，弹出确认对话框说明将有多少章节移入回收站；确认后刷新树结构。
[M] 5. 卷标题行支持拖拽排序，拖拽完成后提交最新顺序并更新 UI。

### AC#2 卷内章节操作

[T] 1. 在指定卷下支持“+ 新建章节”，校验章节标题必填且不超过 50 字。
[M] 2. 章节支持卷内拖拽排序与跨卷拖拽移动，完成后同步更新位置。
[T] 3. 支持 Ctrl / Shift 多选章节，并显示批量移动、批量删除操作栏。
[M] 4. 批量删除执行后，章节从大纲主视图移除并进入回收站视图。

### AC#3 统计信息

[M] 1. 卷标题行右侧展示章节数、卷总字数。
[M] 2. 章节行右侧展示字数、最后编辑时间。
[M] 3. 树形结构底部展示总卷数、总章节数、总字数，并在变更后实时刷新。

### AC#4 与写作工作台章节面板的数据同步

[M] 1. 大纲视图与 US-3.2 章节面板共享同一数据源与更新事件。
[M] 2. 任一侧发生卷章变更后，另一侧实时刷新或增量同步，无需手动重载。
[T] 3. 双击章节标题时直接进入写作工作台并打开对应章节。

## US-1.6 写作目标设定

**依赖**：US-1.4
**前置 Epic**：Epic 1

### AC#1 目标类型

[T] 1. 写作目标 Tab 提供每日字数目标、总字数目标、截止日期输入控件与清除按钮。
[T] 2. 前端校验每日字数 100~50000、总字数 1000~5000000、截止日期必须晚于今天。
[M] 3. 保存时仅提交已设置的字段；清除目标时向后端发送空值或删除指令。

### AC#2 进度展示

[M] 1. 写作工作台底部状态栏根据统计接口显示今日字数进度条、目标值与百分比。
[T] 2. 达成每日目标时播放一次轻量动画（如进度条变绿 + 对勾图标），不使用打断式弹窗。
[M] 3. 项目设置中的写作目标 Tab 展示近 30 天趋势折线图、总进度环形图、预计完成日期。

### AC#3 灵活性

[M] 1. 目标被修改后，状态栏与统计面板立即重算并刷新。
[M] 2. 清除目标后，状态栏不再显示进度条，但历史统计图与统计数据仍保留可见。

## US-1.7 回收站

**依赖**：US-1.5、US-3.2
**前置 Epic**：Epic 1、Epic 3

### AC#1 回收站入口与展示

[T] 1. 项目内回收站入口位于大纲视图底部或写作工作台左侧面板底部。
[M] 2. 打开回收站后按删除时间倒序展示已删除项。
[M] 3. 每条展示标题、类型、所属卷、字数、删除时间、剩余保留天数。

### AC#2 章节删除与恢复

[M] 1. 在 US-1.5 或 US-3.2 删除章节后，前端将章节从正常目录移除，并在回收站中出现对应条目。
[M] 2. 点击恢复时调用恢复接口；若原卷已不存在，则展示“原卷已不存在，已恢复到默认卷” toast。
[M] 3. 恢复成功后，章节与其版本快照的可见状态一起恢复。

### AC#3 保留策略与到期提醒

[M] 1. 回收站列表展示 30 天保留倒计时，并突出显示临近到期项目。
[M] 2. 通知中心收到到期前 3 天提醒后，在回收站和通知列表同步展示提醒状态。
[T] 3. 条目到期或被永久删除后，从列表中即时移除。

### AC#4 手动永久删除

[T] 1. 单条“永久删除”和“清空回收站”都使用二次确认对话框。
[M] 2. 确认后调用永久删除接口，并在成功后刷新列表和相关统计。

### AC#5 存储计算

[M] 1. 回收站页面与存储统计模块展示“回收站占用：{X} MB”。

## US-1.8 项目归档

**依赖**：US-1.4
**前置 Epic**：Epic 1

### AC#1 归档操作

[T] 1. 项目设置危险区和项目卡片菜单都提供“归档项目”入口。
[M] 2. 归档成功后，项目从默认项目列表隐藏，仅在“已归档”筛选下显示。
[M] 3. 进入已归档项目时，编辑器切换只读模式，顶部显示黄色横幅提示先取消归档。

### AC#2 取消归档

[M] 1. 已归档项目的设置页与卡片菜单都提供“取消归档”操作。
[M] 2. 取消归档成功后，项目恢复为可编辑状态并重新进入进行中列表。

### AC#3 数据保留

[T] 1. 归档不会移除正文、知识库、快照、备份等已有数据，前端继续展示对应只读内容。
[M] 2. 自动备份的暂停 / 恢复状态在设置页或状态提示中可见。

## US-1.9 导入已有作品

**依赖**：US-1.3、US-2.1
**前置 Epic**：Epic 1、Epic 2

### AC#1 支持格式

[T] 1. 导入入口支持选择 TXT、DOCX 文件，并在上传前校验单文件不超过 20MB。
[M] 2. 编码自动识别失败时，弹出编码选择器，允许用户手动指定 UTF-8 / GBK / GB2312。

### AC#2 章节智能拆分

[M] 1. 上传成功后进入拆分预览页，左侧展示章节列表，右侧展示原文对照。
[T] 2. 前端提供合并相邻章节、在指定位置拆分、修改章节标题等交互。
[M] 3. 未识别到章节分隔符时，以单章形式展示预览，并提示用户手动拆分。

### AC#3 导入目标

[T] 1. 提供“导入到新项目”“导入到已有项目”两种入口。
[M] 2. 导入到新项目时，复用 US-1.3 创建流程的前两步，最后一步替换为导入确认。
[M] 3. 导入到已有项目时，前端提供目标卷选择器，并在确认后将章节追加到该卷末尾。

### AC#4 导入后衔接 Parser 解析

[M] 1. 导入确认完成后弹出模态对话框，询问是否立即对导入章节运行 AI 解析。
[M] 2. 选择“立即解析”后，前端展示批量解析已入队状态，并在通知中心展示进度。
[T] 3. 解析在后台执行时，导入流程与编辑器保持可操作，不阻塞用户继续使用。

### AC#5 格式处理

[M] 1. 导入预览与结果页保留基础格式：段落分隔、加粗、斜体。
[M] 2. 导入完成后展示导入报告：总字数、章节数、被忽略的格式元素列表。
[T] 3. 对不支持的格式使用说明文本提示，而不是直接阻断导入流程。
