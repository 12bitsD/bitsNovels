# Epic 6 FE 拆分 - 前端聚焦统一设置入口与通知中心交互

## 目标结论
- Epic 6 是系统层能力，独立于写作主链路，可以排在业务 Epic 之后统一交付。
- FE 需要交付一个用户级设置容器，覆盖 `资料 / 偏好 / 快捷键 / 通知与 AI / 存储` 五类 Tab，并保证改动立即生效。
- `US-6.4 通知配置` 与 `US-6.6 通知中心` 必须共用同一套通知类型和分类映射，避免设置页与铃铛面板出现口径不一致。

## FE 边界
- `US-6.1 用户资料管理`：负责资料表单、安全表单、会话列表、危险操作确认 UI。
- `US-6.2 界面偏好设置`：负责主题切换、字体字号行距、音效开关、导出模板管理界面。
- `US-6.3 快捷键配置`：负责分组列表、录入模式、冲突提示、恢复默认、速查浮窗。
- `US-6.4 通知配置`：负责通知类型列表、渠道开关、浏览器授权提示、全部静音、AI 全局偏好表单。
- `US-6.5 存储管理`：负责配额概览、分类图表、分项目明细、清理建议与确认弹层。
- `US-6.6 通知中心`：负责顶部铃铛、未读角标、分类筛选、通知列表、已读/删除/分页交互。

## 信息架构
- 顶层入口是全局用户设置页，左侧为 Tab 导航，右侧为对应内容区。
- 顶部导航常驻通知铃铛，任何页面都可打开通知中心下拉面板。
- 所有设置默认按用户级保存，对该用户所有项目立即生效，不需要刷新页面。

## US-6.1 用户资料管理

### 资料 Tab
- 提供 `昵称`、`笔名`、`头像` 三项编辑能力。
- 头像上传前端先校验 `<= 2MB`，进入裁剪弹窗，固定输出正方形预览。
- 表单有脏值检测；无改动时 `保存` 按钮禁用，提交成功后更新当前登录态展示。

### 账号安全区
- 修改密码使用 `旧密码 + 新密码 + 确认新密码` 的独立表单，前端做基础规则校验和二次一致性校验。
- 第三方账号区域用卡片展示 `Google / GitHub` 绑定状态、最近绑定时间、绑定/解绑按钮。
- 活跃会话列表至少展示 `设备名 / 浏览器 / 最近活跃时间 / IP 区域 / 当前设备标记`，其他设备支持单条注销。

### 危险区域
- 页面底部固定为危险区域卡片，使用强调色和二次确认文案区分普通操作。
- 点击 `注销账号` 后弹出确认对话框，要求输入密码，并明确显示“30 天内可恢复，30 天后永久删除”。
- 若账号处于冻结期，资料页顶部需显示恢复提示条和 `取消注销` 入口。

## US-6.2 界面偏好设置

### 外观设置 UI
- 主题切换使用显式选项卡或卡片：`明亮 / 暗黑 / 护眼 / 跟随系统`。
- 编辑器样式区包含 `字体`、`字号 14-24`、`行距 1.5 / 1.75 / 2.0`，修改后立即反映到当前编辑器。
- 语言项 V1 仅展示 `简体中文`，但保留不可编辑的多语言占位说明，避免后续结构返工。

### 音效设置 UI
- `计时器提示音`、`打字音效` 都使用独立开关，默认关闭。
- 开关切换后立即生效，不要求单独保存；若失败，则回滚控件状态并提示原因。

### 导出模板管理 UI
- 在偏好页内展示来自 `US-5.3` 的模板列表，支持重命名、删除。
- 删除操作需二次确认；重命名使用行内编辑或弹窗都可以，但要保留名称长度提示。
- 这里只负责管理入口和交互，不承担模板内容编辑逻辑。

## US-6.3 快捷键配置

### 列表与分组
- “快捷键”Tab 按 `编辑器操作 / AI 功能 / 面板控制 / 导航` 四组展示。
- 每行展示 `功能名 / 当前快捷键 / 默认快捷键 / 恢复默认`。
- 列表支持键盘可访问性，焦点进入某项后可以直接触发录入模式。

### 录入与冲突提示
- 点击当前快捷键区域后进入捕获态，显示“请按下新的组合键”。
- 若与其他功能冲突，行内报错并指出冲突功能名，不写入本地状态。
- 支持逐条恢复默认和 `全部恢复默认`。

### 快捷键速查
- 编辑器内响应 `Ctrl+/` 或 `Cmd+/`，打开快捷键速查浮窗。
- 浮窗只展示高频快捷键和分组标题，不进入完整设置页也可快速查看。

## US-6.4 通知配置

### 通知类型列表
- “通知与 AI”Tab 先展示通知配置，再展示 AI 全局偏好。
- 每种通知类型独立成行，行内展示 `通知名称 / 说明 / 站内通知 / 浏览器推送`。
- `站内通知` 默认开启且不可关闭，控件在视觉上要明确是锁定态。

### 渠道交互
- 首次开启浏览器推送时，前端先请求系统通知权限；拒绝后保持关闭并提示原因。
- `全部静音` 打开后，所有浏览器推送开关转为关闭态；站内通知继续保留。
- 页面内的通知类型顺序要与通知中心分类映射保持一致，方便用户理解哪些消息会进入哪个分类。

### AI 全局偏好
- 同页提供 `默认模型 / Temperature / 生成长度 / 解析深度` 配置项。
- 表单文案要明确“这是所有项目的默认值，单个项目可覆盖”。
- 保存成功后不需要刷新；下次打开任意项目设置时可读到用户级默认值。

## US-6.5 存储管理

### 概览区
- “存储”Tab 顶部展示总配额、已用空间、使用率进度条。
- 中部展示分类明细图表，类别固定为 `正文 / 知识库 / 快照 / 备份 / 回收站`。
- 使用率超过 `80%` 时，顶部出现黄色提示条。

### 分项目明细
- 项目列表按占用大小降序展示。
- 每个项目支持展开，显示 `正文 / 知识库 / 快照 / 备份` 的细项占用。
- 若项目很多，列表应支持滚动或虚拟化，避免设置页卡顿。

### 清理建议交互
- 智能清理建议列表展示 `建议名称 / 可释放空间 / 查看详情`。
- 点击建议后打开明细弹层，列出将被清理的对象，用户确认后再执行批量清理。
- 存储达到 `100%` 时，前端需要在编辑器保存失败提示中给出明确原因，并提供前往“存储管理”Tab 的快捷入口。

## US-6.6 通知中心

### 顶部铃铛 UI
- 全局顶部导航栏常驻通知铃铛图标。
- 有未读时显示红色数字气泡，超过 `99` 统一显示 `99+`。
- 点击铃铛打开下拉面板；再次点击或点击空白处关闭。

### 分类与列表
- 面板顶部提供 `全部 / AI 解析 / 备份与导出 / 一致性 & 伏笔 / 系统` 五个筛选 Tab。
- 列表项展示 `类型图标 / 标题 / 摘要 / 时间 / 已读状态 / 关联项目名`，未读加粗，已读置灰。
- 列表按时间倒序，支持滚动加载历史，单页 20 条。

### 通知操作
- 单条通知点击后跳转到对应上下文，例如一致性面板、下载链接、存储页。
- 面板内提供 `单条标记已读`、`全部标记已读`、`单条删除`、`清空所有已读`。
- 删除操作只清理通知记录，不影响真实业务数据。

## 前端状态模型
- `profileFormState`：昵称、笔名、头像裁剪结果、脏值、保存状态。
- `securityState`：密码表单、第三方绑定状态、活跃会话列表、冻结期状态。
- `preferencesState`：主题、字体、字号、行距、语言、音效、模板管理列表。
- `shortcutState`：功能分组、当前组合键、冲突信息、录入态、速查浮窗状态。
- `notificationPreferenceState`：通知类型列表、浏览器权限、全部静音、AI 全局偏好。
- `storageState`：总览、分类图表、项目展开项、清理建议、执行中状态。
- `notificationCenterState`：未读数、筛选分类、分页游标、通知列表、跳转目标。
