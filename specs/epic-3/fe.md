# Epic 3 FE 拆分以编辑器交互为主线

## 读者确认
- 核心读者：负责写作工作台的前端工程师与交互设计师
- 已具备背景：已阅读 `specs/EPIC/EPIC3/context.md` 中 Epic 3，并了解 US-1.5、US-6.2 的依赖关系
- 最关心：哪些需求必须在前端实现、哪些状态必须可见、哪些快捷键和交互不能遗漏

## 问题定义
一句话问题：Epic 3 需要拆出可执行的前端验收范围，否则编辑器核心、工作台布局和周边交互容易遗漏。

## 核心结论
1. US-3.1 是前端主干，必须一次覆盖编辑、撤销/重做、保存状态、三栏布局、实时字数统计。 
2. US-3.2、US-3.4、US-3.5、US-3.10 定义了工作台外壳与交互效率，是编辑器可用性的直接门槛。
3. US-3.6 ~ US-3.9 虽然依赖后端存储，但前端仍需负责版本历史、批注、备注、计时器的完整可视化与操作闭环。

## FE Acceptance Criteria

### US-3.1 编辑器核心
结论：前端必须交付一个可长时间写作的富文本编辑器壳层，并完整暴露键盘操作与保存反馈。

- 支持标准输入、删除、选中、复制、粘贴、剪切；富文本工具至少覆盖加粗、斜体、删除线、分隔线、H1~H3。
- **Markdown 实时渲染**：集成 `@tiptap/extension-markdown`，输入 Markdown 语法时自动渲染为富文本格式（类 Typora 体验）。例如输入 `**文字**` 自动渲染为加粗、输入 `## ` 自动升级为 H2 标题、输入 `---` 自动变为分隔线。编辑器内部仍存储富文本 JSON，Markdown 语法是输入快捷方式而非存储格式。
- 支持段落级交互：`Enter` 新段落、`Shift+Enter` 段内换行、`Tab` 缩进；这些操作都进入同一编辑命令体系。
- 粘贴外部富文本时先在前端清洗为受支持格式子集；粘贴纯文本时仅保留段落分隔，不带任何样式。
- 支持 `Ctrl+Z` / `Cmd+Z` 撤销，`Ctrl+Y` / `Ctrl+Shift+Z` / `Cmd+Shift+Z` 重做；撤销栈深度至少 200 步。
- 连续输入需按停顿大于 500ms 或换行为边界合并撤销步骤，避免逐字撤销；自动保存不能清空撤销栈。
- 切换章节或刷新页面时清空当前章节撤销栈；这是前端编辑器实例生命周期的一部分。
- 编辑停止 3 秒后触发自动保存；标题栏必须有 `保存中...`、`已保存 {HH:mm}`、`保存失败 ⚠️` 三种可见状态。
- 保存失败时展示重试入口；连续失败 3 次后弹出模态提示，提醒用户手动复制内容。
- 支持 `Ctrl+S` / `Cmd+S` 立即保存，交互反馈与自动保存一致，但触发时机不等待防抖。
- 页面采用三栏布局：左侧章节面板、中间编辑区、右侧知识库面板；左右栏都支持点击或拖柄折叠，折叠后中栏自适应扩展。
- 编辑区居中，最大宽度保持约 700~800px；底部状态栏实时显示章节字数，以及选区字数。
- 前端实时字数统计只基于正文可见内容，不能把标题样式标记或结构标记算入字数。

### US-3.2 章节管理（写作台侧面板）
结论：左侧章节面板是工作台导航层，前端重点是树形展示、切换、上下文操作和拖拽反馈。

- 以树形结构展示卷章目录，节点展示章节标题、字数、解析状态图标，并高亮当前编辑章节。
- 点击章节节点时切换中间编辑区内容，并同步激活状态到目录树。
- 章节节点右键菜单包含重命名、在上方插入、在下方插入、删除；删除动作在 UI 上明确为移入回收站。
- 支持章节拖拽排序，既可卷内排序，也可跨卷移动；拖拽过程中要有清晰落点提示与成功后的局部刷新。
- 面板底部提供 `+ 新章节` 按钮；创建后默认标题为 `新章节`，自动进入标题编辑态并切换到新章节。
- 面板不显示卷级操作按钮；底部提供 `管理卷结构 ->` 跳转到 US-1.5。

### US-3.3 写作统计
结论：前端需要把统计结果做成可快速展开的写作数据面板，而不是仅返回数字。

- 统计入口位于顶部工具栏或状态栏字数区域，点击后展开详细统计面板。
- 面板展示今日、本周、本月、总字数，以及日均字数、连续写作天数、最高单日记录。
- 可视化至少包含近 30 天柱状图、近 12 周折线图、按小时分布的写作时段热力图。
- 提供按卷、按章节的统计表格，并支持按字数排序。
- 所有统计视图都要处理空数据、加载中和接口失败状态。

### US-3.4 专注模式
结论：专注模式是前端纯交互能力，必须保证界面极简但编辑能力不缩水。

- 支持 `F11` 或工具栏按钮进入专注模式；支持 `Esc` 或顶部悬浮退出按钮退出。
- 进入后隐藏左侧章节面板、右侧知识库面板、顶部菜单栏，仅保留居中的编辑区与低对比度底部状态栏。
- 背景色改为跟随主题的纯色；深色主题下为深灰系，浅色主题下为米白系。
- 自动保存、撤销/重做、保存、查找替换等快捷键在专注模式下保持可用。
- AI 续写、润色类入口不放回固定栏位，改由右键菜单进入。

### US-3.5 编辑器主题
结论：前端主题系统要即时生效，并覆盖颜色、字体、字号、行高四类写作体验参数。

- 提供 Light、Dark、Sepia 三套内置主题，并在用户切换后立即更新工作台视觉。
- 提供系统默认、宋体、黑体、楷体、等宽字体等字体选项。
- 提供 14px~24px 字号调节，以及 1.5 / 1.75 / 2.0 行间距配置。
- 提供 `跟随系统` 选项；启用后监听操作系统深浅色模式并即时切换。
- 主题切换不能破坏当前编辑状态、选区、滚动位置和专注模式状态。

### US-3.6 版本快照
结论：前端重点是版本历史入口、快照列表、Diff 预览和恢复确认流程。

- 工具栏或右键菜单提供 `创建快照` 入口；创建时支持输入不超过 100 字的标签/备注。
- `版本历史` 入口打开侧边面板或模态；列表按时间倒序展示时间、类型、标签、字数。
- 手动快照在列表中使用醒目样式，与自动快照、每日快照明显区分。
- 选中快照后可预览全文，并支持 `与当前版本对比` 的 Diff 视图，增删改需分别标色。
- `恢复到此版本` 必须是显式确认操作，并在恢复成功后刷新正文、字数、保存时间与版本历史列表。
- 自动快照清理、存储占用、删除前确认等后端结果，都需要在前端提供明确反馈。

### US-3.7 内容注释/批注
结论：批注是正文上的增强层，前端要同时处理锚点高亮、浮层、侧栏列表和状态筛选。

- 选中文本后，右键菜单或浮动工具栏提供 `添加批注`；点击后在选区附近弹出输入框。
- 批注输入限制为 500 字；提交后原文以默认淡黄色高亮显示。
- 悬停高亮文字时显示前 100 字预览；点击高亮文字时在右侧展开完整批注详情。
- 右侧 `批注面板` 按文档位置列出当前章节全部批注，并支持定位回正文锚点。
- 每条批注支持编辑、删除、标记已解决；已解决项高亮色变浅灰，并支持筛选显示或隐藏。
- 导出时默认不包含批注，但在导出选项中要有 `包含批注` 开关。

### US-3.8 大纲/章节备注
结论：章节备注是附属于章节的规划层，前端要让备注易写、易收起、易识别，但不干扰正文。

- 在当前章节标题下方提供默认折叠的 `章节备注` 区域；章节面板右键菜单也可进入备注编辑。
- 备注编辑器支持基础富文本能力：加粗、斜体、无序列表。
- 限制备注内容不超过 2000 字，并沿用与正文一致的自动保存反馈。
- 备注默认仅在编辑器内可见，不计入章节字数；正文状态栏与统计视图都不能把备注计入正文产出。
- 章节面板中，有备注的章节显示提示图标。

### US-3.9 写作计时器
结论：计时器是前端浮层工具，必须轻量、可拖拽、不中断写作。

- 工具栏或状态栏提供计时器入口；点击展开非模态浮窗。
- 提供正计时与倒计时两种模式；倒计时默认 25 分钟，允许用户修改分钟数。
- 提供开始、暂停、重置操作，并在浮窗中持续显示当前状态。
- 浮窗支持拖拽定位，不能遮挡主要编辑区域；运行期间编辑器其他功能保持正常。
- 倒计时结束时显示闪烁提示，并在音效未关闭时播放轻柔提示音。
- 计时结束后展示本次写作字数增量，前端需要把开始字数和结束字数结果清晰可见地反馈给用户。

### US-3.10 查找替换
结论：查找替换是高频快捷操作，前端必须把热键、跳转、高亮和撤销语义一次做完整。

- `Ctrl+F` / `Cmd+F` 打开查找栏，`Ctrl+H` / `Cmd+H` 打开替换栏。
- 输入关键词后实时高亮全部匹配项，并展示 `第 {X}/{总数} 个`。
- 支持用上/下箭头按钮或 `Enter` / `Shift+Enter` 在匹配项间跳转。
- 提供 `区分大小写`、`全字匹配` 两项过滤开关。
- `替换` 仅替换当前匹配项，并跳转到下一项；`全部替换` 需要确认后执行，并以 toast 回显替换数量。
- `全部替换` 在前端编辑器命令栈中必须合并成一个撤销步骤；`Esc` 或关闭按钮关闭栏位时要同步清除高亮。
